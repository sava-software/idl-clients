# Fuzz seed corpora

Seed provenance lives here — next to, never inside, a corpus directory, where a
file would itself be fed to the harness as a seed. The replay test that runs
every seed inside `check` is generated by the sava-build hardening plugin
(`ScopeReaderFuzzSeedReplayTest`); the harness contract is documented on
`ScopeReaderFuzz`.

## `scopeReader/`

The fixed 29704-byte `OracleMappings` layout is unreachable from a scratch
mutator, so every seed is either a real account or a minimized finding:

- `4zh6bmb77qX2CL7t5AJYCqa6YqFafbz3QJNeFvZjLowg`, `Chpu5ZgfWX5ZzVpUx9Xvv4WPM75Xd7zPJNDPsFnCpLpk`
  — real mainnet OracleMappings account dumps (file name = account address),
  fetched 2026-07-16. These carry the live oracle-type/refPrice index graph the
  parser's recursion walks.
- `slow-exp-95f9f30f`, `slow-exp-aa041616` — minimized hostile `FixedPrice`
  exponent inputs that drove ~50s parses (`BigDecimal.movePointLeft` inflating a
  very negative scale into a billion-digit `BigInteger`). Fixed by
  `scaleByPowerOfTen` plus Pyth exponent range checks; also pinned by
  `ScopeReaderHostileInputTests`.
